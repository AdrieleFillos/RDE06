Software de Versionamento

  • Sistema que tem por objetivo realizar o controle de versões de um código ou documento.
  • É possível gerenciar as versões de um código ou documento realizadas por um desenvolvedor ou equipe de desenvolvedores.

Quando vai ser desenvolvido um projeto seja codificando um software , site ou a escrita de um documento word de um trabalho acadêmico sobre algum tema , em geral devemos seguir alguns passos que são:

  1. Criar uma pasta;
  2. Criar dentro da pasta o arquivo do projeto que logo deverá ser desenvolvido;
  3. Finalizar e entregar o projeto;

Feita a entrega , pode ser que ocorra a necessidade de alguma correção ou de realizar a atualização de alguma funcionalidade , então para isso , deve-se ser criada uma nova versão do arquivo , realizar as atualizações e entregar.Pode ser que esse processo ocorra por várias atualizações e consequentemente faça com que se crie vários arquivos , causando dificuldades para controlar as versões do projeto. Portano para solucionar esse problema , deve-se utilizar um software que crie um repositório local e crie , a partir do arquivo , um histórico de versões.Além do repositório local , pode-se ser utilizado um repositório remoto , principalmente se o trabalho for em equipe.Neste último caso os arquivos/projetos serão armazenados na nuvem.

  • Um software de controle de versão gerencia tanto o histórico de versão local , quanto o histórico de versão remoto , ou seja , ele sempre vai fazer a sincronização entre esses dois tipos de histórico de versão para que tudo esteja igual.
  • Um software de versionamento controla as alterações permitindo ou não que os usuários façam elas e notifica os usuários.
  • Após a atualização remota , cria-se uma versão corrente que junto com todas as outras versões é distribuída para os usuários sempre cientes das alterações que foram realizadas.

Tipos de Software de Controle de Versão

Centralizado
  • Possui um repositório central em um servidor com controle de versões , ou seja , todos se conectam ao servidor e ficam trabalhando no servidor.
  • Problema: Caso o servidor pare , para-se de controlar as versões e os usuários não conseguem trabalhar.
  • Dependente do servidor.
Distribuído
  • Possui um repositório local com controle de versões.
  • Um usuário pode ficar trabalhando localmente e quando conclui seu trabalho , realiza a atualização no repositório central.
  • Independente do servidor (caso o servidor pare as versões ainda poderão ser controladas).
  • Exemplos: Softwares Mercurial e Git.
  • Git é um software livre de versionamento que surgiu para o desenvolvimento do Kernel do Linux , foi projetado e desenvolvido por Linus Torvalds em 2005.
  • Cada diretório de trabalho do Git é um repositório com um histórico completo e habilidade total de acompanhamento das revisões , não dependente de acesso a uma rede ou a um servidor central.
  • O Github é uma plataforma de hospedagem de código fonte e arquivos com controle de versão.
  • Surgiu em 2008 e foi adquirido em 2018 pela Microsoft por US$ 7,5 bilhôes.
  • 100 milhões de projetos.
  • 56 milhões de usuários.

Comandos GIT

  • A vantagem de utilizar o sistema de direcionamento com a possibilidade de manter repositório local e repositório remoto é a capacidade de fazer alterações de código no projeto em qualquer local.
  • É possível realizar commit dentro da plataforma do Bithub.
  • O pull origin pucha o arquivo para sincronizar a máquina.

Clonando Repositórios

Como fazer uma cópia de um repositório que está remoto:

  1. Entra no Github Desktop.
  2. Clica em add.
  3. Clica em clone repository.
  4. Clica no repositório que quer clonar.
  5. Clica em clone.

Projetos Pré-Existentes

  1. Entra no Github Desktop.
  2. Clica em File.
  3. Clica em New repository.
  4. Escolhe um nome.
  5. Faz a descrição(description).
  6. Marca a opção "initialize this repository with a README"
  7. Clica na licença "MIT License".
  8. Clica em "create repository".
  9. Cola os arquivos do VisualStudio Code na pasta do Github e todos os arquivos vão aparecer no Github.
  10. Faz o commiting e clica em "commiting".
  11. Clica em "publish repository".
